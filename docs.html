Documentation

Complete guide to scanning Solidity smart contracts for security vulnerabilities with VORA.

Installation

VORA runs via npx with no installation required. You need Node.js 18+ installed.

Terminal
$ npx vora scan ./contracts

For frequent use, install globally:

Terminal
$ npm install -g vora $ vora scan ./contracts

Quick Start

Scan a directory of Solidity files:

Terminal
# Scan all .sol files in contracts/ $ npx vora scan ./contracts # Scan with JSON output for CI $ npx vora scan ./contracts --output json # Only show critical and high severity $ npx vora scan ./contracts --severity high # Generate fixes for detected issues $ npx vora scan ./contracts --fix

How It Works

VORA analyzes your Solidity code in three phases:

  1. Parse — Solidity files are parsed into an AST with control flow and data flow graphs
  2. Detect — 200+ static analysis rules run alongside ML models trained on 2,147 real security audits
  3. Report — Vulnerabilities are ranked by severity with detailed context and fix suggestions

Commands

Command Description
vora scan <path> Scan files for vulnerabilities
vora mcp-server Start MCP server for AI agent integration
vora --version Show version
vora --help Show help

Options

Option Default Description
--output, -o text Output format: json, sarif, text
--severity, -s all Filter by severity: critical, high, medium, low, all
--fix, -f false Generate fix suggestions
--pr false Create pull request with fixes (requires --fix)
--exclude, -e — Glob pattern to exclude (e.g., **/*.test.sol)
--quiet, -q false Suppress progress output

Output Formats

JSON

Machine-readable format for programmatic parsing:

JSON Output
{ "version": "0.1.0-beta", "summary": { "total": 3, "critical": 1, "high": 2 }, "findings": [ { "rule_id": "VORA-001", "title": "Reentrancy", "severity": "critical", "location": { "file": "Vault.sol", "line_start": 42 }, "fix": { "description": "Add ReentrancyGuard" } } ] }

SARIF

Static Analysis Results Interchange Format for GitHub/GitLab integration:

Terminal
$ npx vora scan ./contracts --output sarif > results.sarif

Claude Code Integration

Add VORA as an MCP server for Claude Code:

Terminal
$ claude mcp add vora -- npx vora mcp-server

Then in Claude Code, you can ask:

  • "Scan my contracts folder for critical vulnerabilities"
  • "Run a security audit on Vault.sol and fix any reentrancy issues"
  • "Explain the VORA-001 vulnerability"

Cursor Integration

Add to your Cursor MCP configuration at ~/.cursor/mcp.json:

~/.cursor/mcp.json
{ "servers": { "vora": { "command": "npx", "args": ["vora", "mcp-server"] } } }

Usage in Cursor chat: @vora scan ./contracts

VS Code Extension

Install the VORA VS Code extension:

Terminal
$ code --install-extension vora.vora-security

Available commands:

  • VORA: Scan Current File — Scan the active .sol file
  • VORA: Scan Workspace — Scan all Solidity files
  • VORA: Apply Fix — Apply suggested fix at cursor

VORA-001: Reentrancy

VORA-001 Critical

Detects cross-function and cross-contract reentrancy vulnerabilities, including read-only reentrancy patterns.

Vulnerable Pattern
function withdraw(uint256 amount) external { require(balances[msg.sender] >= amount); (bool success, ) = msg.sender.call{value: amount}(""); require(success); balances[msg.sender] -= amount; // State update AFTER call }
Fixed Code
function withdraw(uint256 amount) external nonReentrant { require(balances[msg.sender] >= amount); balances[msg.sender] -= amount; // State update BEFORE call (bool success, ) = msg.sender.call{value: amount}(""); require(success); }

VORA-002: Flash Loan Attacks

VORA-002 Critical

Identifies price manipulation vectors through flash loan-enabled atomic transactions.

Fix Strategy
  • Use TWAP (Time-Weighted Average Price) oracles
  • Add price deviation checks
  • Implement flash loan guards
  • Add minimum holding period for price-sensitive operations

VORA-003: Oracle Manipulation

VORA-003 Critical

Detects vulnerable oracle dependencies and spot price usage without TWAP protection.

Fix Strategy
  • Use multiple oracle sources with aggregation
  • Add freshness checks (heartbeat)
  • Implement circuit breakers for extreme deviations
  • Use Chainlink or other decentralized oracles

VORA-004: Access Control

VORA-004 High

Missing or incorrect access control modifiers on privileged functions.

Vulnerable Pattern
function setFee(uint256 newFee) external { fee = newFee; // Anyone can change fees! }
Fixed Code
function setFee(uint256 newFee) external onlyOwner { require(newFee <= MAX_FEE, "Fee too high"); fee = newFee; }

VORA-005: Integer Overflow

VORA-005 High

Arithmetic operations without SafeMath (pre-0.8.0) or unchecked blocks causing overflow.

Fix Strategy
  • Upgrade to Solidity 0.8.0+ for automatic overflow checks
  • Use SafeMath for Solidity < 0.8.0
  • Remove unchecked blocks unless proven safe
  • Add explicit bounds checking

VORA-006: Front-Running

VORA-006 Medium

Transaction ordering dependencies and sandwich attack vectors in DEX integrations.

Fix Strategy
  • Add slippage tolerance parameters
  • Use commit-reveal schemes for sensitive operations
  • Add transaction deadlines
  • Consider private mempools or Flashbots

FAQ

What Solidity versions are supported?

VORA supports Solidity 0.4.0 through 0.8.x. Version-specific checks (like SafeMath for pre-0.8) are automatically applied.

How accurate is the ML detection?

The ML model was trained on 2,147 real security audits and 12,891 documented vulnerabilities. Detection rates vary by plan:

  • Developer tier: 47% medium, 15% high/critical
  • Auditor tier: 78% medium, 47% high/critical

Can I use VORA in CI/CD without an API key?

Yes, the Developer tier is free up to 5K tokens. For larger projects, set VORA_API_KEY.

How do I integrate with my AI coding assistant?

VORA provides an MCP server that works with Claude Code, Cursor, and other MCP-compatible tools. See the integrations section.

Need help? Check the Agent API docs for programmatic integration, or open an issue on GitHub.
